
**O que é um Banco de Stage (Staging Area)?**

Imagine que você vai cozinhar um prato complexo. Você não joga os ingredientes direto da sacola do supermercado na panela, certo? Primeiro, você os coloca na bancada, lava, descasca e pica. O *Banco de Stage* é essa "bancada da cozinha".

É um ambiente temporário onde os dados são depositados logo após serem extraídos das fontes originais (ERPs, CRMs, Planilhas, APIs).

* **Objetivo Principal:** minimizar o impacto nos sistemas de origem. Você extrai tudo de uma vez para o Stage e, a partir daí, faz o trabalho pesado sem "atrapalhar" o sistema que a empresa usa para vender ou operar.
* **Características:** 
	* Os dados costumam ter a mesma estrutura da origem (sem transformações complexas ainda).
	* Geralmente é um banco de limpeza rápida (os dados não ficam lá para sempre).
	* Não possui índices complexos ou chaves estrangeiras rígidas, para priorizar a velocidade de escrita.
