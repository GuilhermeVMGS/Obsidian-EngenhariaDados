
O **Data Lake** tradicional é apenas um local de armazenamento de arquivos. Ele é ótimo para guardar volumes massivos de dados barato, mas sofre com falta de organização, gerando o famoso "Pântano de Dados".

O **Delta Lake** não é um local de armazenamento separado, ele é uma *camada de armazenamento de código aberto* que roda sobre o seu Data Lake. Ele possui um **log de transações** (*Delta Log*) que traz funcionalidades de bancos de dados relacionais para arquivos comuns. Ou seja, ele acrescente características ACID (Atomicidade, Consistência, Isolamento e Durabilidade) aos Data Lakes, trazendo uma diminuição de arquivos duplicados. Um Delta Lake comun, usa tabelas que correspondem a níveis de qualidade que adicionam estrutura aos dados (estrutura medalhão).
