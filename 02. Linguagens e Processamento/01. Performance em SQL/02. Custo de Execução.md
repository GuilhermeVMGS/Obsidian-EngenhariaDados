
Antes de rodar qualquer query, o banco de dados possui um componente chamado **Otimizador**. Ele avalia vários caminhos possíveis para entregar o resultado e escolhe o de "menor custo"
* **O Plano de Execução:** é o mapa que o otimizador cria. Ele decide se vai usar um índice, se vai fazer um scan completo ou como vai juntar (*Join*) duas tabelas.
- **Estatísticas:** o custo é calculado com base em estatísticas (quantas linhas a tabela tem, quão variados são os dados em uma coluna). Se as estatísticas estiverem desatualizadas, o banco pode escolher um caminho lento achando que é o mais rápido.
- **Fatores de Custo:** o "custo" geralmente engloba:
    1. **I/O (Entrada/Saída):** ler dados do disco (o fator mais caro).
    2. **CPU:** processar cálculos, ordenações e filtros.    
    3. **Memória:** espaço para realizar operações temporárias.
