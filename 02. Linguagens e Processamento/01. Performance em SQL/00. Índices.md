
Um índice é uma estrutura de dados separada da tabela principal. Ele funciona exatamente como um índice remissivo de um livro técnico.

* **Conceito:** ao invés de ler todas as linhas de uma tabela (*Full Table Scan*), o banco consulta o índice para descobrir em quais endereços de memória a informação está.
* **A Estrutura B-Tree:** a maioria dos índices usa essa estrutura. Ela permite que o banco encontre um registro em pouquíssimos passos, dividindo a busca sucessivamente.
* **O Custo:** os índices aceleram a leitura (*SELECT*) mas tornam a escrita (*INSERT, UPDATE, DELETE*) mais lenta, pois o banco precisa atualizar o índice toda vez que os dados mudam.
