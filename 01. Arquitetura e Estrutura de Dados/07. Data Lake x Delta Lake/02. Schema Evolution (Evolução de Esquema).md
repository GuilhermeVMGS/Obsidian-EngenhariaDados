
Imagine que uma tabela sempre teve 5 colunas, mas depois de um tempo uma 6a coluna precisou ser adicionada. No Data Lake comum, isso costuma quebrar processos.

No **Delta Lake** temos algumas opções extras:

* *Schema Enforcement (Imposição):* O Delta Lake impede que dados com colunas "estranhas" entrem na tabela por erro, protegendo a qualidade.
- *Schema Evolution (Evolução):* É o recurso que permite que você mude a estrutura da tabela intencionalmente. O Delta Lake atualiza os metadados para aceitar a nova coluna sem que você precise reescrever todos os dados antigos.
