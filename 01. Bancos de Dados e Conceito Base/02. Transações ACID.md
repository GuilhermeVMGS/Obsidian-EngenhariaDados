
	ACID: Atomicidade, Consistência, Isolamento, Durabilidade.

Uma transação (comando) em um banco de dados é uma operação tratada como uma unidade de trabalho indivisível. Isso significa que a transação é concluída integralmente ou não ocorre, garantindo a consistência do sistema. Um exemplo é a retirada de dinheiro de uma conta bancária: ou o valor é sacado com sucesso ou a operação não acontece. Não há estado intermediário.

Uma transação em um banco de dados é um bloco lógico de uma ou mais instruções SQL que devem ser executadas todas juntas ou nenhuma.

**ACID** são as principais características que definem uma transação.

* **Atomicidade:*** cada instrução de uma transação (leitura, gravação, atualização ou exclusão) é tratada como um única unidade: ou a operação acontece inteira, ou não acontece nada. Sem dados "pela metade".
* **Consistência:** garante que os dados sigam regras de esquema definido. As transações apenas modificam as tabelas de maneiras predefinidas e previsíveis. A consistência transacional mantém as tabelas consistentes em caso de corrupção de dados ou erros, evitando resultados de execução não intencionais.
* **Isolamento:** vários usuários podem ler e gravar na mesma tabela ao mesmo tempo, mas as transações são isoladas para que as simultâneas não interfiram ou afetem umas às outras.
* **Durabilidade:** garante que as alterações de dados feitas por transações executadas com sucesso sejam preservadas, mesmo em caso de falha do sistema. Ou seja, após uma transação ser confirmada com sucesso, as alterações são permanentes e sobrevivem a falhas de hardware ou software.
