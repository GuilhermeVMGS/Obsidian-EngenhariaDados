
Na linha do tempo, o **Data Warehouse** surgiu antes do **Data Lake**, que foi criado como resposta a novas necessidades.
O DW não conseguia mais atender sozinho às novas demandas dos dados, então o DL veio para resolver problemas que surgiram com Big Data, cloud e novos tipos de dados.

**Antes do Data Lake, o cenário era:**

1. Apenas Dados Estruturados
*DW foi feito para*
* Tabelas relacionais, fatos e dimensões, dados bem definidos.
*Problema:*
* Logs, JSON, XML, Texto, Imagens, Dados de Sensores.

2. Schema-on-write (estrutura anterior)
*No DW:*
* Precisa-se saber como o dado será usado
* Define o modelo antes de carregar
*Problema:*
* Perguntas novas podem surgir depois
* Mudanda de schema é cara
* Muito retrabalho com ETL

3. Custo alto de armazenamento
* DWs tradicionais eram caros
* Armazenar dados "talvez úteis" não valia a pena
*Resultado:*
* Muito dado era descartado ou nem chegava a ser armazenado

4. Baixa flexibilidade para exploração
*DW é ótimo para:*
* KPIs
* Relatórios conhecidos
*Ruim para:*
* Exploração
* Data Science
* Machine Learning
* Análises ad hoc não planejadas

**Com isso, o DL resolveu os seguintes pontos:**

1. Armazenar tudo (primeiro, perguntar depois)
* Dados brutos
* Qualquer formato
* Sem schema prévio (schema-on-read)

2. Custo baixo e escala
* Storage barato (HDFS, S3, ADLS)
* Escala quase infinita

3. Flexibilidade analítica
* BI
* Data Science
* ML
* Streaming
* Reprocessamento histórico

4. Separação de responsabilidades
* Data Lake = ingestão + histórico bruto
* Data Warehouse = dado curado e confiável

Resumo: o DL foi criado para permitir o armazenamento barato, escalável e flexível de grandes volumes de dados brutos e variados, algo que o DW tradicional não conseguia fazer de forma eficiente.
