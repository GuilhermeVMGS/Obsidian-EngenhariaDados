
	ACID: Atomicidade, Consistência, Isolamento, Durabilidade.

Uma transação (comando) em um banco de dados é uma operação tratada como uma unidade de trabalho indivisível. Isso significa que a transação é concluída integralmente ou não ocorre, garantindo a consistência do sistema. Um exemplo é a retirada de dinheiro de uma conta bancária: ou o valor é sacado com sucesso ou a operação não acontece. Não há estado intermediário.

**ACID** são as principais características que definem uma transação.

* **Atomicidade:** todas as instruções de uma transação são executadas como um bloco indivisível. Se uma falhar, todo o conjunto é desfeito, evitando dados corrompidos ou parciais.
* **Consistência:** a transação leva o banco de dados de um estado válido para outro estado válido, respeitando todas as regras e restrições predefinidas, sem criar inconsistências.
* **Isolamento:** transações concorrentes não interferem umas nas outras, cada uma é executada como se fosse a única, mantendo os dados consistentes para todas as operações simultâneas.
* **Durabilidade:** após uma transação ser confirmada com sucesso (commit), as alterações são permanentes e sobrevivem a falhas de hardware ou software.
