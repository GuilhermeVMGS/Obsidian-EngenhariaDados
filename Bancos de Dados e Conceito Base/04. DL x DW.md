
**Data Warehouse** é um repositório centralizado de dados estruturados, organizado para análise de relatórios.

Suas principais características são:
* Dados estruturados
* Modelo definido previamente (schema-on-write)
* ETL forte (transforma antes de carregar)
* Alta qualidade e governança
* Otimizado para OLAP

Tipos de dados:
* Vendas
* Financeiro
* Indicadores consolidados
* Métricas Históricas

Exemplos de uso:
* BI
* Relatórios gerenciais
* KPIs corporativos

**Data Lake** é um repositório de dados brutos, que armazena dados em seu formato original.

Suas principais características são:
* Dados estruturados, semiestruturados e não estruturados.
* Schema-on-read (estrutura definida na leitura)
* Carga rápida (ELT ou apenas ingestão)
* Flexível e escalável

Tipos de dados:
* Logs
* JSON, XML
* Dados de sensores
* Imagens, textos
* Dados brutos de sistemas OLTP

Exemplo de uso:
* Big Data
* Machine Learning
* Exploração de dados
* Base para múltiplos DWs

| Aspecto       | Data Warehouse (DW)          | Data Lake (DL)            |
| ------------- | ---------------------------- | ------------------------- |
| Surgimento    | Mais antigo                  | Mais recente              |
| Tipo de dado  | Estruturado                  | Qualquer tipo             |
| Estrutura     | Schema-on-write              | Schema-on-read            |
| Transformação | Antes da carga (ETL)         | Na leitura (ELT)          |
| Governança    | Alta                         | Menor (se não bem gerido) |
| Uso principal | BI / Relatórios              | Exploração / ML / staging |
| Performance   | Alta para queries analíticas | Depende do motor          |
